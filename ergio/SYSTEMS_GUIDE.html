1. What ERGIO Actually Is

ERGIO is an AI Business Operating System. It does two things no other platform does together:

  1. BUILD — Generates a complete business from a text prompt (website, brand, CRM, lead forms, booking, invoices)
  2. OPERATE — Runs the business after launch (finding leads, booking appointments, sending invoices, collecting payments, creating content, answering calls)

Think of it as: Lovable (website builder) + GoHighLevel (marketing ops) + Clay (lead gen) + Cal.com (booking) + Paystack (payments) — all from one prompt, all connected, all automated.

The one-liner: "Everyone else builds apps. ERGIO builds businesses that run themselves."

2. System Architecture — 4 Layers

ERGIO is built in 4 distinct layers, each with a specific job:

Layer 1: Frontend (Vercel)

Static HTML/CSS/JS — no framework, no build step. Serves the landing page, dashboard, website builder, admin panel, and all user-facing pages. Vercel handles CDN, SSL, and edge deployment globally.

  • Vercel Edge deployment + CDN
  • Supabase Auth Login/signup via Firebase-compatible auth
  • engines-api.js JavaScript client that talks to the Render backend

Layer 2: AI Engine Backend (Render)

Python FastAPI server — the "brain" of ERGIO. No timeout limit (unlike Vercel's 10s). Handles all AI generation, lead scraping, conductor chat, MCP calls, plugin execution, and skill management.

  • FastAPI Async Python web framework
  • Groq Primary LLM (Llama 3.3 70B)
  • OpenRouter Backup LLM (multi-model)
  • Pollinations Image generation fallback

Layer 3: Data Layer (4 Databases)

  • Supabase (PostgreSQL) — Structured data: users, businesses, leads, bookings, invoices, payments
  • MongoDB — Unstructured AI content: generated websites, marketing content, brand assets
  • Firebase — Real-time features: live chat, notifications, booking status
  • Redis — Task queue (Celery), event bus (Pub/Sub), caching, rate limiting

Layer 4: Integration Layer (MCP + Plugins + Agent Frameworks)

  • 36 MCP Servers — External tool connections via Model Context Protocol
  • 30 Plugins — App-specific integrations (WhatsApp CRM, Instagram, TikTok, etc.)
  • 4 Agent Frameworks — External AI agents (Hermes, OpenClaw, Composio, Glama)

3. The Complete User Flow — Start to Finish

1
User arrives at landing page → sees the prompt box: "Describe your business idea." The hero section showcases what ERGIO can build. No login required to start.
2
User types a prompt → e.g., "I'm a lawyer in Lagos specializing in property law. I need clients." → clicks "Start Building Free."
3
Build animation starts → ERGIO shows a live build overlay with stages: Planning → Brand Identity → Website Content → SEO → Final Assembly. This is visual feedback while the AI works.
4
AI generates the website → The Conductor (on Render) receives the prompt, calls the Website Engine, which:
  • Plans the business structure (pages, services, pricing)
  • Generates brand identity (name, tagline, colors, logo via Pollinations)
  • Writes all website copy (hero, about, services, contact)
  • Optimizes for SEO (meta tags, keywords, schema.org data)
  • Assembles full HTML/CSS/JS website
5
User sees live preview → The generated website appears in an iframe. User can refine it via chat ("make the header blue", "add a pricing section") — each refinement is an AI call that modifies the HTML.
6
User creates account → Supabase auth creates a profile. The website is saved to the user's account in Supabase (generated_websites table).
7
User enters dashboard → Sees their business with 10 engine panels:
  • Leads tab → Lead Engine starts scanning the internet for potential clients
  • Bookings tab → Booking Engine creates a calendar and booking form
  • Invoices tab → Invoice Engine generates PDF-ready invoices
  • Payments tab → Payment Engine connects Paystack for collections
  • Conductor tab → Chat with ERGIO AI for any task
8
ERGIO starts operating → While the user manages their business, ERGIO works in the background:
  • Lead Engine finds people needing legal services → scores them 0-100 → WhatsApp alerts for hot leads (80+)
  • Content Engine generates social media posts → user approves → ERGIO posts
  • When a lead books → Invoice Engine auto-generates invoice → Payment Engine sends payment link
  • After service → Reputation Engine sends review request → routes to Google or private feedback
9
Analytics Engine compiles data → Weekly insights report: revenue, leads, conversion rate, customer acquisition cost. User sees everything in the dashboard.

4. The 10 Engines

1. Website Engine BUILD

What it does: Generates a full business website from a text prompt.

How it works: Takes the user's prompt → sends to Groq (Llama 3.3 70B) with a structured prompt that asks for: business plan, brand identity (name, tagline, color scheme), website content (hero, about, services, pricing, contact), and SEO metadata. AI returns JSON → ERGIO assembles it into a complete HTML/CSS/JS website with images from Pollinations. The website is saved to Supabase.

2. Lead Engine OPERATE

What it does: Finds people actively looking for the user's service.

How it works: Uses SearXNG to scan 70+ search engines (Google, Bing, DuckDuckGo, etc.) for queries like "need a lawyer in Lagos" or "property dispute help Nigeria." Scrapes results with Cheerio (HTML parser). AI scores each lead 0-100 based on: urgency, budget signals, location match, service match. Hot leads (80+) trigger WhatsApp alerts. Data stored in Supabase leads table.

3. Booking Engine OPERATE

What it does: Manages calendar and appointment booking.

How it works: Connects to Cal.com MCP for calendar sync. Creates a booking form on the user's website. When someone books → saves to Supabase bookings table → sends WhatsApp confirmation → schedules a reminder 1 hour before. Integrates with Google Calendar via MCP.

4. Invoice Engine OPERATE

What it does: Auto-generates invoices after booking.

How it works: When a booking is confirmed → generates an invoice with: business name, client name, service description, amount, due date, payment link. Sends via Resend (email) and WhatsApp (via Twilio). Tracks payment status in Supabase invoices table. Sends follow-up reminders for overdue invoices.

5. Payment Engine OPERATE

What it does: Collects payments via Paystack, Flutterwave, and Stripe.

How it works: Integrates Paystack's API (initialize → verify → webhook). When a client clicks "Pay" on an invoice → Paystack processes the payment → webhook hits ERGIO's /payments endpoint → ERGIO verifies the payment → marks invoice as paid → sends receipt → updates revenue dashboard. Supports cards, bank transfer, USSD, mobile money.

6. Content Engine OPERATE

What it does: Generates marketing content (social posts, blog posts, ad copy, video scripts).

How it works: Uses Groq to generate platform-specific content: Twitter (280 chars), Instagram (caption + hashtags), Facebook (longer post), LinkedIn (professional tone). Also generates blog posts for SEO and video scripts for Higgsfield AI (video generation). Human approves before publishing — all content goes to the Approval Gateway.

7. Voice Engine OPERATE

What it does: AI voice receptionist answers calls and books appointments.

How it works: Connects to Vapi AI or Retell AI (MCP servers). When a call comes in → AI answers with a custom greeting → understands the caller's intent → books appointment (via Cal.com MCP) → answers FAQs (using business data from Supabase) → can also send WhatsApp voice messages via ElevenLabs. Calls are recorded and transcribed for quality.

8. Analytics Engine OPERATE

What it does: Tracks revenue, leads, conversion rates, and generates weekly insights.

How it works: Aggregates data from all Supabase tables: payments (revenue), leads (acquisition), bookings (conversion). Calculates: customer acquisition cost (CAC), lifetime value (LTV), conversion rate (leads → booked → paid). Generates a weekly report using AI. Revenue forecasting based on pipeline data (leads in progress × historical conversion rate × average deal size).

9. Reputation Engine OPERATE

What it does: Manages reviews and online reputation.

How it works: After a service is completed (booking marked as "completed") → sends a WhatsApp review request. If the client rates 4-5 stars → routes them to Google Business Profile to leave a public review. If 1-3 stars → routes them to a private feedback form. Monitors new reviews across Google, Facebook, and other platforms. Alerts the user when a negative review appears.

10. Workflow Engine OPERATE

What it does: Visual drag-and-drop automation builder.

How it works: Users create if/then workflows: "When lead score is 80+, send WhatsApp alert + create invoice." Powered by n8n MCP. Pre-built workflow recipes for common patterns: lead follow-up sequence, invoice reminder chain, review request automation. Workflows are stored in Supabase and executed via Redis task queue.

5. The Conductor — AI Agent Runtime

The Conductor is the "brain" of ERGIO. It's an AI agent that lives on Render and handles all user requests via chat (like Claude Code).

Architecture:

User Chat → Conductor → Decides which Engine to activate
                          → Calls MCP if external tool needed
                          → Calls Plugin if app integration needed
                          → Checks Human Approval Gateway
                          → Returns result to user

Components:

  • MCP Client — Connects to all 36 MCP servers, manages connections, handles reconnection
  • Plugin Sandbox — Executes plugin code in isolated containers, prevents crashes
  • Task Queue — Redis + Celery, async job execution with retry logic
  • State Manager — Tracks each business in Supabase, maintains conversation context
  • Human Approval Gateway — Pauses external actions (social posts, invoice sends, payments) and sends to dashboard for approval
  • Event Bus — Redis Pub/Sub for inter-engine communication (lead captured → workflow triggered)
  • Circuit Breaker — If an MCP or plugin fails 3 times, it's disabled and alerts the user
  • Memory System — 3 layers: Durable Facts, Procedural Skills, Session Search

How the Conductor decides:

  1. User types: "Find me clients for my law firm"
  2. Conductor analyzes intent → determines this is a Lead Engine task
  3. Conductor calls the Lead Engine with the user's business context
  4. Lead Engine runs SearXNG search, scrapes results, AI scores leads
  5. Results returned to Conductor → Conductor presents them to user
  6. Conductor asks: "Want me to send cold emails to the top 5?"
  7. If user says yes → Conductor calls Content Engine to write emails → checks Approval Gateway → user approves → sends via Resend

6. MCP System — 36 Protocol Servers

MCP (Model Context Protocol) is the standard that lets AI agents talk to external tools. ERGIO's Conductor acts as an MCP Client — it sends requests to MCP Servers and receives structured responses.

Categories:

  • Payments (3): Stripe, Paystack, Flutterwave
  • Communication (5): Twilio, Resend, Postmark, WhatsApp Business, Slack
  • AI (4): Groq, OpenRouter, Pinecone, Higgsfield
  • Voice & Caller (8): Vapi AI, Retell AI, Bland AI, Synthflow, AgentCall, AgentPhone, SkipCalls, GreetMate
  • Agent Frameworks (4): Hermes Agent, OpenClaw, Composio, Glama
  • Marketing (3): Google Ads, Meta Ads, Mailchimp
  • Productivity (4): Notion, GitHub, Google Maps, Cal.com
  • Infrastructure (3): Cloudflare, Vercel, Sentry
  • Others (4): Apify, Instantly, n8n, Tavily

How MCP works:

ERGIO Conductor (MCP Client)
    ↓ sends: {"tool": "send_whatsapp", "params": {"to": "+234...", "msg": "..."}}
MCP Server (Twilio)
    ↓ executes API call
Twilio API
    ↓ returns: {"status": "sent", "message_id": "..."}
MCP Server
    ↓ returns structured response
ERGIO Conductor
    ↓ presents result to user

7. Plugin System — 30 Plugins

Plugins are app-specific integrations that give ERGIO direct access to external platforms. Unlike MCPs (which are tool servers), plugins are direct API integrations that execute actions on specific platforms.

Categories: Social Media (8), Communication (7), Payments (3), Productivity (4), Analytics (2), Marketing (3), CRM (1), E-commerce (1), Design (1).

8. Skills Marketplace

Like Claude Code Skills — users upload custom instruction sets that teach ERGIO how to handle specialized tasks. The Conductor auto-discovers relevant skills for each task and loads them dynamically.

7 built-in skills ship with ERGIO: Nigeria Business Expert, Yoruba/Igbo/Hausa Marketing, Paystack Expert, WhatsApp Business, Restaurant Operations, Law Firm Management, Nigeria SEO Specialist.

Users can upload their own skills, rate them 1-5 stars, and download community skills. Skills are stored in the Skills Marketplace API on the Render backend.

9. External Agent Frameworks

ERGIO can connect to external AI agent systems via MCP. This means ERGIO can delegate tasks to specialized agents that have their own intelligence:

Hermes Agent (Nous Research)

Self-improving AI agent. Key feature: it gets better over time. It remembers what worked, writes new skills from successful interactions, and has its own memory system. ERGIO connects to Hermes via its MCP server — sends a task → Hermes uses its own LLM + memory + tools → returns result.

OpenClaw

Multi-agent framework. You can spin up specialized agents (research agent, coding agent, data agent) and orchestrate them. Each agent has its own memory and tool set. ERGIO's Conductor can delegate to specific OpenClaw agents based on the task type.

The System Behind It

All agent connections use the Model Context Protocol (MCP). The formula: AI Agent = LLM + Memory + Tools + Planning + Feedback. The LLM provides reasoning. Memory provides context. Tools provide capability. Planning provides strategy. Feedback provides improvement. ERGIO's Conductor orchestrates all of this.

10. Security System — 10 Layers

  1. Rate Limiting — 100 req/min default, 10/min for auth endpoints. Prevents brute force.
  2. XSS Prevention — Blocks <script>, javascript:, onerror= patterns in all inputs.
  3. SQL Injection Detection — Scans for UNION SELECT, DROP TABLE, OR 1=1 patterns.
  4. Content Security Policy (CSP) — Restricts what scripts can run. Only allows ERGIO's own domains.
  5. CORS — Only ergio.vercel.app and the Render URL are allowed origins.
  6. IP Blocklist — Auto-blocks IPs after 5 failed auth attempts.
  7. Payload Size Limit — Max 1MB per request. Prevents DoS attacks.
  8. HSTS — Forces HTTPS on all connections. Prevents man-in-the-middle.
  9. X-Frame-Options DENY — Prevents clickjacking (nobody can embed ERGIO in an iframe).
  10. API Key Hashing — All API keys are SHA-256 hashed. Never stored in plain text.

11. Database Architecture

ERGIO uses 4 databases, each with a specific job:

  • Supabase (PostgreSQL) — 23 tables: profiles, businesses, leads, bookings, invoices, payments, services, clients, reviews, transactions, engine_status, pending_approvals, activity_logs, outreach_log, plugin_configs, workflows, subscriptions, generated_content, memory_facts, memory_skills, session_memory, whatsapp_conversations, referrals. Row-level security enabled.
  • MongoDB — Unstructured AI content: full generated website HTML, marketing content, brand assets, MCP responses.
  • Firebase — Real-time: auth (already working), live chat messages, booking status updates, notification delivery.
  • Redis — Task queue (Celery jobs), event bus (Pub/Sub for inter-engine communication), caching, rate limiting.

12. AI Provider Router

ERGIO has a 3-tier AI fallback system:

  1. Groq (Primary) — Llama 3.3 70B. Fastest inference (sub-second). Free tier (14K tokens/min). Best for website generation and lead scoring.
  2. OpenRouter (Backup) — Multi-model access (GPT-4, Claude, Llama, Mistral). Used when Groq is rate-limited.
  3. Pollinations (Fallback) — Free, no API key, always available. Used for image generation and as ultimate text fallback.

The router automatically falls back: if Groq fails → tries OpenRouter → if that fails → uses Pollinations. This ensures ERGIO never goes down due to AI provider issues.

13. Payment Flow

1. Invoice Engine generates invoice → saves to Supabase
2. Payment Engine creates Paystack payment link
3. Client receives link via email (Resend) + WhatsApp (Twilio)
4. Client clicks link → Paystack checkout page
5. Client pays (card/bank transfer/USSD/mobile money)
6. Paystack sends webhook to ERGIO /api/payments
7. ERGIO verifies payment with Paystack API
8. Invoice marked as "paid" in Supabase
9. Receipt sent to client
10. Revenue dashboard updates in real-time
11. Analytics Engine records the transaction
